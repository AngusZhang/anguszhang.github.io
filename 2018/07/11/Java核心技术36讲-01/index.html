Java核心技术36讲_01

Posted on 2018-07-11 Edited on 2021-01-18 In Java

  • 问题提出

    谈谈你对 Java 平台的理解？“Java 是解释执行”，这句话正确吗？

  • 问题分析

    这种问题属于比较笼统的问题，目的是问了了解你对 Java 平台的理解，并从回答中可以找到继续深入的方向，类似的问题还有谈谈你对 Spring 的理解，这种问题的答案可以片面，但是要尽量深入，所以可以从你熟悉的方向入手引导面试官。

  • 方向

    1. 从 Write once, run anywhere 的宣传与可以引申到 Java 是如何获得跨平台的能力的，从而谈论下对 JVM（类加载、垃圾收集）、字节码、JIT 等的理解。与此同时对“Java 是解释执行”这句话提出自己的理解。
    2. 从集合类库出发，包括但不仅限于包括集合、IO/NIO、网络、并发、安全在内的基础类库。

  • 一张图

  • 需要明确的概念

    1. Javac 的编译，编译 Java 源码生成“.class”，里面实际是字节码，而非机器码。执行方式可以通过 JVM 直接解释或者便以为目标机器码。
    2. 解释、编译、混合、分层编译与 AOT，这些概念后面的课程应该会讲到，到时另有文章补充。
