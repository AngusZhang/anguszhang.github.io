将应用从 Spring 4 迁移到 Spring Boot 2

Posted on 2021-01-19 Edited on 2021-01-20 In 实战

背景

随着业务规模逐渐扩大，项目需要向微服务架构转型，将现有应用迁移到 SpringBoot，为之后的服务化做准备。

版本

旧项目使用的 Spring 版本为 4.3.x，刚开始迁移时选择了 SpringBoot 当时最新 2.3 版本，但是由于工程已经比较庞大，且引入了一些比较旧的三方包，导致版本兼容比较困难，在不断尝试下最终决定将 SpringBoot 版本定在 2.1.17.RELEASE，SpringFramework 版本定在 5.1.18.RELEASE。

以上的版本仅供参考，具体的版本选择需要根据具体情况而定。

总结

本文总结了一下笔者迁移到 SpringBoot 时的大致流程，写这篇文章的时候服务已经上线两个多月，运行正常。相信还有很多朋友们的应用由于各种原因仍然使用着比较老的框架（不仅限于Spring）

  1. 选择合适的版本减少改造既有工程的代价
  2. 做好面对枯燥工作的心里准备
  3. 安排好充分的测试
  4. 最重要是领导同意（狗头）

放手去做，会有好结果的。
